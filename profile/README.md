Welcome to the **OpenBeavs** GitHub organization! We are an Oregon State University (OSU) CS 46X Capstone team dedicated to building accessible, intelligent tools like the **GENESIS AI Hub**.

🔗 **Live instance:** [genesis.dev.oregonstate.edu](http://genesis.dev.oregonstate.edu)

## 🚀 About Us

OpenBeavs focuses on creating modern software solutions for the OSU community and beyond. Our capstone projects span full-stack web development, Retrieval-Augmented Generation (RAG) pipelines, and integrating AI into functional, user-facing applications. 

## 📂 Featured Repositories

- 🦫 **[OpenBeavs](https://github.com/OpenBeavs/OpenBeavs)**
  The core OSU OpenBeavs application developed for our CS 46X Capstone.
- 🧠 **[OSU-RAG-Pipeline](https://github.com/OpenBeavs/OSU-RAG-Pipeline)**
  A RAG (Retrieval-Augmented Generation) pipeline built in Python to empower our AI knowledge base.

## 🛠️ Tech Stack & Focus

Our development work prominently features:
- **Languages:** JavaScript, Python
- **AI & Data:** RAG architectures, Python backend pipelines
- **Auth & Integration:** Microsoft SSO integrations
- **Web:** Modern JS frameworks, Open WebUI

## 📁 Archived Repositories

As our project evolves, some repositories are archived to preserve their history and learnings while we refine our active core frontend and backend systems. Here is a look at our foundational past work:

- 🚪 **[Microsoft-SSO](https://github.com/OpenBeavs/Microsoft-SSO)**
  *Insight:* Served as our initial proof-of-concept for securely handling Oregon State University authentication flows. It provided foundational knowledge on integrating MSAL (Microsoft Authentication Library) seamlessly before migrating the finalized authentication logic into the main app ecosystem.
  
- ⏪ **[original-frontend](https://github.com/OpenBeavs/original-frontend)**
  *Insight:* The very first iteration of our user interface. This repository captures our early design choices, rapid prototyping, and component structures. The lessons learned here were critical before we transitioned to our current, more robust UI architecture integrated with our RAG pipeline.

- 🌐 **[open-webui](https://github.com/OpenBeavs/open-webui)**
  *Insight:* A customized fork and exploration of the Open WebUI framework. This repository allowed us to deeply understand the deployment, customization, and constraints of open-source LLM interfaces, which heavily informed the final UX/UI decisions for the GENESIS AI Hub.
